## Bug 1 (P0): CLI Arguments Never Effective

### Problem

The `if __name__ == "__main__":` block at the end of `sse2json.py` contained a
full copy of the startup logic from `main()` — `_load_router_state()`,
`_prefetch_model_summaries()`, `_start_state_autosave()`,
`_start_health_score_updater()`, `_start_model_discovery_queue()`, server
creation, and `serve_forever()`. This meant `main()` was never called, and all
CLI arguments parsed by `main()` (`--host`, `--port`, `--config`, `--log-dir`,
etc.) were silently ignored.

### Fix

Replaced the entire `if __name__ == "__main__":` block with a single
`main()` call. This ensures CLI arguments are parsed and applied.

### Files

- `sse2json.py`: Replaced ~40 lines of duplicated startup code with `main()`.

## Bug 2 (P1): Helper Functions Use Global Runtime Instead of Request Snapshot

### Problem

The `RuntimeContext` bundle was introduced to swap all five globals
(`CONFIG`, `ROUTER`, `UPSTREAM_CLIENT`, `OBSERVABILITY`, `AUDIT`) as one atomic
reference. Request handlers capture this snapshot at entry via
`_request_runtime()`.

However, module-level helper functions still read the globals directly:

- `_record_failed_attempt` → global `OBSERVABILITY`
- `_record_upstream_http_failure` → global `ROUTER`
- `_record_request_conversion_failure` → global `ROUTER`
- `_record_transport_failure` → global `ROUTER` + global `CONFIG`
- `_record_proxy_exception` → global `ROUTER`
- `_record_stream_interrupted` → global `ROUTER`
- `_record_empty_visible_output_failure` → global `ROUTER`
- `_request_json_once_with_timing` → global `UPSTREAM_CLIENT`
- `_request_raw_once_with_timing` → global `UPSTREAM_CLIENT`
- `_open_stream_with_compat_retry` → global `UPSTREAM_CLIENT`
- All compat retry functions → global `OBSERVABILITY.record_first_byte`

During a config hot-reload (`_apply_runtime_config`), these helpers could use a
**new** router/observability instance while the request's attempt was created by
the **old** router. This caused:

- `report_failure` updating the wrong `_keys_state` (cooldown/failure tracking corruption)
- `record_attempt` writing to the wrong observability instance (observability data loss)
- `record_first_byte` recording latency against the wrong observability instance

### Fix

Added thread-local runtime storage:

```python
_request_rt = threading.local()

def _set_request_rt(rt: RuntimeContext) -> None:
    _request_rt.ctx = rt

def _clear_request_rt() -> None:
    _request_rt.ctx = None

def _current_rt() -> RuntimeContext:
    rt = getattr(_request_rt, "ctx", None)
    if rt is not None:
        return rt
    return _request_runtime()
```

The three request handlers (`_proxy_openai_chat_completions`,
`_proxy_responses`, `_proxy_anthropic_messages`) now call
`_set_request_rt(rt)` immediately after capturing the runtime snapshot.

All helper functions use `_current_rt()` instead of globals. The fallback to
`_request_runtime()` ensures admin/background paths (which don't set the
thread-local) still work correctly.

### Files

- `sse2json.py`: Added thread-local runtime storage; updated 10+ helper functions.

## Bug 3 (P1): Reasoning/Thinking Detection Uses Global CONFIG

### Problem

`_chat_upstream_requires_reasoning_content` and
`_anthropic_upstream_requires_thinking` read the global `CONFIG` to check
provider-level `force_reasoning_content` / `force_anthropic_thinking` flags.
During a hot-reload, these functions could read a new config while the request
was started with an old config.

### Fix

Both functions now use `_current_rt().config` instead of global `CONFIG`.

### Files

- `sse2json.py`: 2 functions updated.

## Bug 4 (P1): resolve_model Uses Global MODEL_MAP/DISABLE_MAP

### Problem

`resolve_model` read the module-level globals `MODEL_MAP` and `DISABLE_MAP`
directly. These globals are refreshed by `_refresh_model_mapping_globals()`
during `_apply_runtime_config()`, creating a window where `MODEL_MAP` could
be from the new config while the caller's `CONFIG` is from the old config (or
vice versa).

### Fix

When `config` is passed (request handler path), `resolve_model` now reads
`client_model_map` and `disable_client_model_map` directly from the config
dict, bypassing the global variables entirely. When no config is passed
(tests, admin paths), it falls back to the global `MODEL_MAP` / `DISABLE_MAP`
for backward compatibility.

### Files

- `sse2json.py`: `resolve_model` updated; backward-compatible with test patches.

## Bug 5 (P2): Prefetch Thread Pool Reference Count Leak

### Problem

`prefetch_initial_stream_lines` in `stream_adapters.py` called
`_get_prefetch_pool()` which increments `_PREFETCH_POOL_USAGE_COUNT`. The
function never called `_release_prefetch_pool()` to decrement it. The counter
only increased, so the prefetch thread pool was never recycled even when all
prefetch operations completed successfully.

### Fix

Added `_release_prefetch_pool()` call in a `finally` block after the
`_get_prefetch_pool().submit()` call, ensuring the counter is decremented on
all paths (success, timeout, exception).

### Files

- `stream_adapters.py`: 2 lines added (`finally: _release_prefetch_pool()`).

## Bug 6 (P2): Connection Pool Not Cleaned on Exit

### Problem

The `KeyboardInterrupt` handler in `main()` called `_save_router_state()` and
`s.server_close()` but did not call `UPSTREAM_CLIENT.close()`, leaving HTTP
connection pools (urllib3 `PoolManager` instances) open. This could leak
sockets and file descriptors on shutdown.

### Fix

Added `UPSTREAM_CLIENT.close()` before `s.server_close()` in the
`KeyboardInterrupt` handler.

### Files

- `sse2json.py`: 1 line added.

## Bug 7 (P2): Duplicated Startup Code

### Problem

The `if __name__ == "__main__":` block duplicated ~40 lines of startup logic
from `main()`, including:
- `_load_router_state()`
- `_prefetch_model_summaries()`
- `_start_state_autosave()`
- `_start_health_score_updater()`
- `_start_model_discovery_queue()`
- Server creation and configuration
- `serve_forever()` with `KeyboardInterrupt` handling

This was the root cause of Bug 1 (CLI args never effective) and also created
a maintenance burden: any change to startup logic had to be applied in two
places.

### Fix

Replaced the entire `if __name__ == "__main__":` block with `main()`.

### Files

- `sse2json.py`: ~40 lines removed, replaced with single `main()` call.

## Other Improvements

### admin_routes.py: Model Pricing Cache Lock

Added `_MODEL_PRICING_LOCK = threading.Lock()` to prevent thundering-herd when
the model pricing cache expires. Multiple concurrent dashboard poll threads
would all miss the cache, all build the fast_lookup index, and all write the
cache simultaneously. With the lock, only one thread does the work; others wait
and reuse the result.

### model_registry.py: Enhanced Model Discovery

Improved model discovery logic for better provider model tracking.

### usage_accounting.py: Usage Statistics Improvements

Improved token usage normalization and cost estimation.
